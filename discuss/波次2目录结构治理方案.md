# 波次 2 目录结构治理方案

更新时间：2026-06-09  
适用范围：W2-I 目录结构治理评审稿  
当前口径：波次 2 约 90% 完成，剩余约 10%，本方案用于进入最后一段结构治理前确认边界。

---

## 1. 背景

项目规则要求：

- Python / TypeScript 等动态语言单文件尽量不超过 300 行。
- 每层文件夹中的文件尽量不超过 8 个。
- Run & Debug 操作必须通过 `scripts/` 下的 `.sh` 脚本进行。
- 讨论和评审用方案写入 `discuss/`。

当前波次 2 已完成 W2-0、W2-A、W2-B、W2-C、W2-D、W2-E、W2-F、W2-G、W2-H。剩余 W2-I 不应再新增业务能力，而应收束架构坏味道，降低后续波次 3 的维护成本。

---

## 2. 当前超限点

| 目录 / 文件 | 当前状态 | 问题 |
|---|---:|---|
| `scripts/` | 43 个文件 | 启停、检查、初始化、任务脚本混在根目录，查找成本高 |
| `docs/` | 17 个文件 | 编号文档多，但存在大量历史引用，直接搬迁风险高 |
| `src/reflexlearn/orchestration/nodes/` | 11 个文件 | 节点职责开始分化，后续元认知/协作深化会继续膨胀 |
| `src/reflexlearn/rag/` | 10 个文件 | 检索后端、融合、路由、ACL 混在一层 |
| `tests/unit/flow/` | 9 个文件 | `test_metacognition.py` 已超 300 行，且 flow 测试开始拥挤 |

---

## 3. 治理目标

W2-I 只做结构治理，目标如下：

1. 拆分超 300 行测试文件。
2. 让关键目录回到“每层尽量不超过 8 文件”的结构。
3. 保留全部现有业务行为和 API 契约。
4. 保留根目录 `scripts/*.sh` 兼容入口，避免破坏项目 Run & Debug 铁律。
5. 对 `docs/` 先做索引和后续归档规划，不在本轮大搬迁编号文档。

---

## 4. 明确禁止事项

本轮 W2-I 不做以下事情：

- 不新增业务 API。
- 不修改现有 API 请求/响应结构。
- 不改变 eval 分数口径。
- 不改变前端路由。
- 不重写 RAG、LangGraph、错题飞轮、LoRA 导出逻辑。
- 不删除历史文档。
- 不破坏 `bash scripts/*.sh` 的根目录可调用方式。
- 不把真实训练、部署、Safety Gateway、HttpOnly Cookie 等波次 3 内容混入本轮。

### 5.4 第四优先级：治理 `scripts/`

原因：

- 当前 `scripts/` 43 个文件，是最大超限点。
- 但项目铁律要求所有 Run & Debug 入口都通过 `scripts/` 下 `.sh` 使用，不能一次性把入口挪走。

建议采用“两层兼容入口”，但必须先完整盘点现有 43 个脚本，不能只搬迁清单里的子集。

```text
scripts/
  _lib.sh
  start_api.sh
  stop_api.sh
  start_frontend.sh
  stop_frontend.sh
  test_unit.sh
  build_frontend.sh
  check_api.sh
  checks/     # 检查脚本真实实现
  jobs/       # 离线/评测/消费任务真实实现
  init/       # 初始化脚本真实实现
  ops/        # 启停中间件脚本真实实现
  probes/     # 探测脚本真实实现
```

但是本轮建议只做第一步：

- 新建 `scripts/checks/`、`scripts/jobs/`、`scripts/init/`、`scripts/ops/`。
- 把低风险脚本内部实现移动到子目录。
- 根目录保留同名 `.sh` 包装入口。
- 子目录真实实现不能继续假设 `_lib.sh` 在同目录。必须使用如下兼容写法：

```bash
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
SCRIPTS_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
source "$SCRIPTS_ROOT/_lib.sh"
```

- 根目录包装入口仍可使用 `source "$SCRIPT_DIR/_lib.sh"`，并 `exec "$SCRIPT_DIR/<subdir>/<script>.sh" "$@"`。
- 所有兄弟脚本调用必须改为通过 `SCRIPTS_ROOT` 或根入口调用。例如 `check_metacognition_perf.sh` 里的 `bash "$SCRIPT_DIR/run_eval.sh"` 搬入子目录后必须变成 `bash "$SCRIPTS_ROOT/run_eval.sh"` 或 `bash "$SCRIPTS_ROOT/jobs/run_eval.sh"`。

包装入口示例：

```bash
#!/usr/bin/env bash
set -euo pipefail
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
exec "$SCRIPT_DIR/checks/check_lora_export.sh" "$@"
```
