# 登录后学习系统首页与信息架构设计方案

## 1. 设计结论

登录后的第一个界面命名为 **Today / 今日学习**。

它不是后台首页，也不是功能入口列表，而是学习系统的每日驾驶舱。用户进入系统后，第一眼应该知道：

1. 今天应该继续什么学习目标。
2. 系统为什么建议我做这一步。
3. 我可以立刻从哪里开始行动。

核心表达：

> ReflexLearn 帮学习者决定下一步怎么学，并把每一次学习反馈写回计划。

## 2. 当前问题

当前登录后的页面已经有左侧导航和多个功能页，但体验仍偏框架：

- 进入后缺少一个真正的默认首页。
- 左侧导航列出了功能，但没有告诉用户先做什么。
- 页面之间缺少学习闭环关系，像独立模块，而不是一个系统。
- “学习路径”“资源库”“成长档案”等页面有 MVP，但入口和状态不够聚合。
- 用户没有看到“系统已经理解我、正在推进我”的感觉。

因此需要先重构信息架构，再重构页面视觉。

## 3. 产品核心闭环

登录后系统围绕一条学习闭环组织：

```text
目标输入
  -> 学习画像
  -> 今日建议
  -> 学习路径
  -> 资源生成 / AI 辅导
  -> 错题复盘
  -> 成长档案
  -> 下一轮建议
```

Today 首页承担闭环的聚合与调度，不替代各功能页。

## 5. Today 首页信息架构

### 5.1 第一屏目标

第一屏要让用户立刻进入学习状态，而不是浏览功能。

推荐布局为 **上方状态栏 + 三列主区域 + 下方最近动态**。

### 5.2 顶部区域

内容：

- 问候语：`今天继续推进哪一步？`
- 当前主学习目标：例如 `前端面试冲刺 / 机器学习入门`
- 当前学习空间状态：进行中、已学习天数、今日建议数
- 用户画像摘要：当前重点、薄弱点、学习偏好

设计要求：

- 不使用大面积营销标题。
- 标题要短，像产品内页，而不是公开官网。
- 主按钮清晰，例如 `继续学习`、`生成今日计划`。

### 5.3 主区域：三列结构

#### 左列：继续学习

用途：承接用户上次离开的地方。

内容：

- 当前学习空间卡片。
- 上次学习位置。
- 今日推荐完成的 2-3 个小任务。
- 主操作：`继续上次学习`。

空状态：

- 如果没有学习空间，展示“创建你的第一个学习目标”。
- 输入框可以直接写学习目标，例如“准备前端面试”。

#### 中列：下一步建议

用途：这是 Today 首页的核心。

内容：

- 系统推荐的下一步。
- 推荐理由：来自画像、路径、错题、资源状态中的哪一项。
- 预计耗时。
- 可选动作：
  - 开始 AI 辅导
  - 生成学习路径
  - 生成资源
  - 复盘错题

设计要求：

- 这一列视觉权重最高。
- 不要堆很多卡片，只放最重要的一条建议和少量备选建议。
- 文案要像学习教练，而不是系统日志。

#### 右列：快速入口

用途：给用户低成本开始动作。

内容：

- 输入新学习目标。
- 上传学习资料。
- 录入错题。
- 生成学习路径。
- 打开资源库。

设计要求：

- 使用明确动词。
- 每个入口都应该说明会产生什么结果。
- 不要只放图标菜单。

### 5.4 下方区域

#### 最近资源

展示最近生成或保存的资源：

- 文档
- 导图
- 题库
- 代码实操
- 阅读材料
- 视频脚本

入口指向 `/resources`。

#### 错题提醒

展示：

- 待复习错题数量。
- 最近错题归因。
- 推荐补救资源。

入口指向 `/mistakes`。

#### 成长变化

展示：

- 本周学习次数。
- 薄弱点变化。
- 画像更新摘要。
- Agent 最近更新了什么建议。

入口指向 `/growth`。

## 6. 导航规划

左侧导航建议调整顺序：

1. Today
2. AI 学习助手
3. 学习空间
4. 学习路径
5. 资源库
6. 知识库
7. 错题复盘
8. 成长档案

原因：

- Today 是默认入口。
- AI 学习助手是最强行动入口。
- 空间、路径、资源、知识库构成学习资产。
- 错题和成长档案构成反馈闭环。

不建议继续使用“工作台”作为主表达。可以在技术或路由内部保留 workspace，但面向用户的文字尽量使用“学习系统”“学习空间”“今日学习”。

## 7. 功能连接关系

### 7.1 Today 到各页面

| Today 模块 | 点击后进入 | 行为 |
| --- | --- | --- |
| 继续学习 | `/chat` 或目标空间详情 | 继续上次会话或目标 |
| 今日建议 | `/chat`、`/plan`、`/mistakes` | 按建议类型进入 |
| 当前学习空间 | `/spaces` 或空间详情 | 查看目标资产 |
| 最近资源 | `/resources` | 查看资源 |
| 错题提醒 | `/mistakes` | 复盘错题 |
| 成长变化 | `/growth` | 查看画像和轨迹 |
| 上传资料 | `/knowledge` | 上传文档 |

### 7.2 数据来源

第一版可优先复用已有接口和本地聚合：

| 数据 | 可用来源 | 第一版策略 |
| --- | --- | --- |
| 用户信息 | `/auth/me` | 已有 |
| 学习空间 | `/spaces` | 可接真实数据 |
| 资源 | `/resources` | 可接真实数据 |
| 错题 | `/mistakes` | 可接真实数据 |
| 成长/轨迹 | `/growth/traces` | 可接但需转译成人话 |
| 学习路径 | 对话生成结果 | 第一版可做空状态和引导 |
| 画像摘要 | 现有 profile 链路间接存在 | 第一版可先用示例/摘要占位 |

## 8. 视觉设计方向

Today 页面应比当前列表页更像成熟产品：

- 页面背景延续暖白纸张，但加入轻微分区层次。
- 左侧深蓝导航可以保留，但需要降低“后台系统”感。
- 主区域采用清晰的栅格和信息权重。
- 卡片数量减少，改成少量高质量模块。
- 关键操作按钮使用统一深蓝色。
- 状态、标签、进度使用克制颜色，不做花哨渐变。
- 避免大面积圆角卡片堆叠，卡片只用于具体功能块。

## 9. 首页空状态设计

如果用户没有任何学习数据，Today 不应该空白。

推荐空状态：

1. 中央展示“创建你的第一个学习目标”。
2. 输入框示例：
   - `三周内准备前端面试`
   - `从零学习机器学习`
   - `复盘高等数学错题`
3. 提供三个快捷动作：
   - 上传资料
   - 录入错题
   - 让 AI 生成学习路径

空状态的目标是让用户 10 秒内开始第一步。

### 10.1 立即可做

1. 新增 `Today` 页面。
2. 调整左侧导航，加入 Today，并让登录后默认进入 Today。
3. Today 页面接入：
   - `/spaces`
   - `/resources`
   - `/mistakes`
   - `/growth/traces`
4. 做空状态和有数据状态。
5. 将现有页面的入口文案统一为“学习系统”语境。

### 10.2 需要后端补强后再做

1. 学习空间详情页。
2. 独立学习路径列表/详情 API。
3. 完整画像摘要 API。
4. 差距报告 API。

## 11. 开发顺序建议

### 第一步：产品骨架

- 新增 Today 页面。
- 调整导航顺序。
- 登录后默认进入 Today。
- 用现有接口填充 Today 的基础数据。

### 第二步：学习空间详情

- 每个学习空间有详情页。
- 聚合路径、资源、错题、知识库和成长记录。

### 第三步：学习路径产品化

- `/plan` 从说明页变成路径管理页。
- 支持当前路径、阶段节点、推荐理由、下一步动作。

### 第四步：成长档案重构

- `/growth` 从轨迹 JSON 页面改成画像和成长趋势页面。
- 协作轨迹作为“系统如何更新建议”的二级内容。

### 第五步：课程、导师、差距报告

- 在基础闭环稳定后补课程中心、导师匹配和行业差距报告。

## 12. 需要确认的问题

1. 登录后默认路由是否直接改为 `/today`？
2. 侧边栏是否保留深蓝背景，还是改成更轻的纸张/墨色导航？
3. Today 第一版是否允许使用画像摘要占位，等画像 API 稳定后替换？
4. 是否优先做学习空间详情页，让 Today 的“继续学习”有更完整落点？
