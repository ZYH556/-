## 1. 当前结论

ReflexLearn 当前已经具备可重复运行的评测闭环：

- 默认机器学习评测集：`ml-001` 到 `ml-005`。
- 规则 Judge：无 LLM key 时稳定评估 correctness / profile_match / completeness / format_quality / overall。
- LLM-as-a-judge 接口：有 key 时走 LLM，无 key 或输出异常自动降级规则 Judge。
- 消融策略：`full-smoke` / `no_rag` / `no_reflexion` / `controlled_rag` / `controlled_reflexion` / `single_agent_baseline`。
- 元认知 smoke 策略：`metacognition_off` / `metacognition_on`，用于显式验证开关接线与 trace 是否进入 `metacognition` 节点。
- 元认知真实策略：`metacognition_real_off` / `metacognition_real_on`，保留 LLM key、关闭 RAG/多轮/内部 planner 以隔离变量，用于验证 self-refine 真实链路与阶段收益。
- 真实评测策略：`real_full` / `real_no_rag` / `real_no_reflexion`，由 `--real` 或 `scripts/run_real_eval.sh` 进入。
- 真实评测入口已加入评测专用收敛配置：按 `EvalCase.expected_resource_types` 规划资源、固定 central、关闭内部 LLM Profile/Planner/质量检查、跳过评测无关 path_plan，并在 assemble 后停止收集，避免把规划/路径排序耗时混入资源质量消融。
- `real_full` / `real_no_reflexion` 增加 RAG preflight；Qdrant collection 不可用时输出 `rag_preflight_failed`，不进入 LangGraph，也不伪造真实 RAG 结论。
- 本轮已完成真实 RAG 小样本活体：在 Graph/Bigdata/Observe 服务可用、Qdrant collection 已初始化、reranker 重模型默认关闭的条件下，`real_full` 不再 blocked。
- 报告输出：JSON + Markdown，写入 `logs/eval_report.*` 或 `logs/eval_comparison.*`。
- 报告会标注 `Judge 来源`，避免把规则降级结果误读成真实 LLM judge。
- 资源覆盖率指标：`resource_coverage` 用于衡量生成资源类型是否覆盖 case 期望。
- OpenAI-compatible timicc 中转已实测通过：`OPENAI_COMPAT_WIRE_API=responses` 时走 `https://timicc.com/responses`，`scripts/check_llm.sh` 返回合法 JSON。

当前最有说服力的已验证结论：

| 对比项 | 结论 |
|---|---|
| 多资源编排 vs 单 Agent doc-only | 多资源编排资源覆盖率为 100.0%，单 Agent baseline 为 33.3%。 |
| 真实 full RAG vs no_rag vs 单 Agent baseline | `real_full` 在 `ml-004` 上 coverage 100.0%、overall 0.6900；`real_no_rag` coverage 100.0%、overall 0.5467；`single_agent_baseline` coverage 33.3%、overall 0.2000。 |
| RAG 受控上限 vs no_rag | `controlled_rag` 在 `ml-004` 上 correctness 从 0.0000 提升到 1.0000。 |
| Reflexion 受控上限 vs no_reflexion | `controlled_reflexion` 在 `ml-005` 上 correctness 从 0.2000 提升到 1.0000。 |
| 元认知 smoke on/off | `metacognition_on` 在 `ml-005` trace 进入 `metacognition` 节点；快速 smoke 为稳定清空 LLM key，元认知自评降级跳过，on/off 当前 overall 均为 0.7058。 |
| 元认知带 key on/off | `metacognition_real_on` 在 `ml-004` trace 进入 `metacognition` 节点并二次生成，`overall=0.6408`；`metacognition_real_off overall=0.5733`。但 on 策略耗时到 90s，Judge 部分走规则降级，因此只能作为阶段证据。 |

## 3. 指标定义

| 指标 | 含义 |
|---|---|
| `task_completion_rate` | case 是否至少生成资源，且命中至少一种期望资源类型。 |
| `resource_coverage` | 生成资源类型覆盖期望资源类型的比例。 |
| `correctness` | 资源内容对 reference concepts 的覆盖率。 |
| `profile_match` | 资源难度是否落在 case 的难度区间内。 |
| `completeness` | 内容长度与目标覆盖的规则评分。 |
| `format_quality` | Markdown / 题目答案 / 代码块等格式提示评分。 |
| `overall` | 加权综合分：correctness 0.35、profile_match 0.20、completeness 0.30、format_quality 0.15。 |

## 5. 靶向消融结果

### 5.1 RAG 依赖 Case：`ml-004`

命令：

```bash
bash scripts/run_eval.sh --compare --tags ablation,rag_required \
  --strategies no_rag,controlled_rag,single_agent_baseline --max-cases 1 --timeout 12
```

| Strategy | resource_coverage | correctness | overall | 解释 |
|---|---:|---:|---:|---|
| `no_rag` | 100.0% | 0.0000 | 0.6058 | 离线多资源能生成，但没有命中 Qdrant / BM25 / Neo4j / RRF / ACL。 |
| `controlled_rag` | 100.0% | 1.0000 | 0.9495 | 注入固定参考知识后，核心概念全部命中。 |
| `single_agent_baseline` | 33.3% | 0.0000 | 0.5525 | 只生成 doc，资源覆盖不足，且未命中 RAG 概念。 |

结论：在 RAG 依赖 case 上，知识上下文是否存在会显著影响 correctness；当前受控 RAG 基线证明了“有正确检索上下文”的上限收益。

### 5.2 Reflexion 依赖 Case：`ml-005`

命令：

```bash
bash scripts/run_eval.sh --compare --tags ablation,reflexion_required \
  --strategies no_reflexion,controlled_reflexion,single_agent_baseline --max-cases 1 --timeout 12
```

| Strategy | resource_coverage | correctness | overall | 解释 |
|---|---:|---:|---:|---|
| `no_reflexion` | 100.0% | 0.2000 | 0.6463 | 多资源能生成，但仅部分命中失败修复概念。 |
| `controlled_reflexion` | 100.0% | 1.0000 | 0.9521 | 注入失败归因、修复策略、重规划和质量校验经验后，核心概念全部命中。 |

结论：在失败修复 case 上，显式 Reflexion 经验能明显提升 correctness 与 overall。

## 7. 当前限制

当前报告中的 `controlled_rag` / `controlled_reflexion` 是受控上限基线，不是生产链路真实召回结果：

- `controlled_rag` 不连接真实 Qdrant / Neo4j / BM25 索引。
- `controlled_reflexion` 不连接真实长期记忆库。
- 默认环境无 LLM key 时，Judge 会降级为规则 Judge；本机已验证 OpenAI-compatible timicc key 可触发真实 LLM judge。
- 当前仍缺少扩大样本后的真实 RAG / Reflexion 统计结论和人工抽检；小样本 LLM judge 可作阶段证据，但不能替代最终比赛论据。
- 元认知真实对比目前只跑 1 条 `rag_required` case；`metacognition_real_on` 因 self-refine 额外外呼导致耗时接近 90s，评分阶段出现 `judge_timeout_rule_fallback`，不能宣称稳定收益，只能说明真实链路可运行且阶段分数为正。
- `real_full` / `real_no_rag` 已在 180s 小样本窗口跑通；60s 仍可能受中转站慢请求影响，不建议作为正式真实评测窗口。
- `scripts/run_real_eval.sh` 默认关闭 reranker 重模型；需要验证 bge-reranker 精排收益时显式设置 `ENABLE_RERANK=true`，并避免与 API/单测同时加载大模型。
- `full-smoke` 为保证速度关闭慢依赖，因此它不是最终完整系统能力上限。

这些限制不影响当前阶段用途：评测靶场、指标、报告、策略对比链路已经跑通；后续可直接替换为真实 RAG / Reflexion 环境。
