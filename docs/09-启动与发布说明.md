# 启动与发布说明

本文档用于接手开发、评审演示和 GitHub 发布前检查。所有运行、调试、测试操作必须通过 `scripts/` 目录下的脚本完成。

## 一、环境准备

### 1. 基础工具

- Python 使用 `.venv` 虚拟环境。
- Python 依赖管理使用 `uv`。
- 前端依赖由 `frontend/package-lock.json` 锁定。
- 本地服务日志统一输出到 `logs/`。

### 2. 环境变量

首次启动前复制配置：

```bash
cp .env.example .env
```

开发演示可使用 `.env.example` 中的默认演示账号；生产环境必须修改以下值：

```text
APP_ENV
AUTH_SECRET_KEY
AUTH_DEMO_PASSWORD
CORS_ALLOW_ORIGINS
TRUSTED_HOSTS
DATABASE_URL
REDIS_URL
QDRANT_URL
```

生产环境下后端会拒绝默认密钥、默认演示密码以及关闭鉴权的配置。

## 二、本地启动

### 1. 启动核心依赖

```bash
bash scripts/start_core.sh
```

核心依赖主要覆盖 PostgreSQL、Redis、Qdrant 等基础服务。

### 2. 初始化数据

```bash
bash scripts/init_all.sh
```

该脚本负责数据库、向量库和内置知识样例初始化。

### 3. 启动 API

```bash
bash scripts/start_api.sh 8000
```

API 地址：

```text
http://localhost:8000
```

健康检查：

```text
http://localhost:8000/api/health
```

### 4. 启动前端

```bash
bash scripts/start_frontend.sh 3000 http://localhost:8000/api
```

前端地址：

```text
http://localhost:3000
```

默认演示登录：

```text
用户名：admin
密码：reflexlearn-admin
```

### 5. 停止服务

```bash
bash scripts/stop_frontend.sh 3000
bash scripts/stop_api.sh 8000
bash scripts/stop_all.sh
```

## 三、验证命令

### 1. 单元测试

```bash
bash scripts/test_unit.sh
```

### 2. 前端构建

```bash
bash scripts/build_frontend.sh http://localhost:8000/api
```

### 3. API 安全冒烟

```bash
bash scripts/check_api_security.sh 8000
```

检查范围包括：

- health 接口可用；
- auth login 可用；
- 受保护路由没有 Bearer token 时会被拒绝；
- 上传 guard 的基础校验；
- 视频任务提交的鉴权边界。

### 4. API 集成真写

```bash
bash scripts/check_api_integrations.sh 8000
```

该脚本依赖 Qdrant/PostgreSQL 等中间件可用，用于验证知识上传真写和视频降级链路。

### 5. Shell 脚本语法检查

```bash
bash -n scripts/*.sh
```

## 四、GitHub 发布前检查

发布前必须确认以下事项：

1. `.env` 不进入 Git。
2. `.venv/` 不进入 Git。
3. `frontend/node_modules/` 不进入 Git。
4. `logs/` 不进入 Git。
5. `__pycache__/`、`.pytest_cache/` 等缓存不进入 Git。
6. README 已说明项目能力、启动方式和已知限制。
7. 运行过至少一轮单元测试和前端构建。

## 五、推荐发布流程

如果项目目录已经初始化 Git，并且已配置 GitHub remote：

```bash
git status -sb
git add .gitignore README.md docs/09-启动与发布说明.md
git commit -m "补充项目介绍与启动发布说明"
git push -u origin main
```

如果项目还不是 Git 仓库：

```bash
git init
git branch -M main
git add .gitignore README.md docs/09-启动与发布说明.md
git commit -m "初始化 ReflexLearn 项目文档"
git remote add origin <你的 GitHub 仓库地址>
git push -u origin main
```

## 六、后续自动更新建议

当前项目还没有配置生产 CI/CD。推送到 GitHub 以后，可以按下列路径逐步升级：

1. 先用 GitHub 作为源码主线，所有升级通过 `git pull` 和 `git push` 同步。
2. 再补 GitHub Actions，自动运行 `scripts/test_unit.sh`、`scripts/build_frontend.sh` 和脚本语法检查。
3. 最后接入 Vercel、Render、Docker 主机或自有服务器，由 main 分支触发自动构建和部署。

在 CI/CD 完成前，不能把 GitHub push 等同于生产自动发布。
