## 3. 已完成能力

### 3.1 多智能体学习资源生成

主链路位于 `src/reflexlearn/orchestration/graph.py`：

```text
START
→ profile
→ recall
→ planner
→ generate_resource 或 pipeline
→ gate
→ critic 或 debate/judge 或 assemble
→ path_plan
→ END
```

已经实现的 Agent/节点职责：

- `profile`：基于对话上下文生成学习画像。
- `recall`：召回反思记忆和相关上下文。
- `planner`：把用户学习目标拆成学习资源任务。
- `generate_resource`：按任务类型调用资源生成技能。
- `gate`：质量验收，不达标进入反思或辩论。
- `critic`：失败归因并推动重规划。
- `debate/judge`：对冲突方案做多方辩论和裁决。
- `assemble`：把通过验收的资源组装成资源包。
- `path_plan`：生成可解释的学习路径。

### 3.2 学习资源类型

资源生成技能位于 `src/reflexlearn/skills/`，当前支持：

- 文档讲解
- 测验题

- 视频脚本/分镜

### 4.3 建议测试重点

优先测：

1. 未登录访问 `chat / knowledge / video` 是否被拒绝。
2. 登录后 token 是否能正常调用接口。
3. 上传非法类型、超大文件、伪造 MIME 是否被拦截。

## 6. 主要风险和坏味道

### 6.1 生产安全仍不完整

当前是 P0 最小安全底座，不是生产安全闭环。最重要的缺口是对象级权限、审计、上传隔离和 AI Safety Gateway。

### 6.2 前端仍是演示工作台

前端不是完整产品信息架构。缺少会话管理、知识库列表、资源库、管理台、用户设置、系统状态、错误恢复等页面。

### 6.3 评测结论不能过度宣传

规则 judge 和 smoke/受控消融只能证明管线可跑，不能直接证明最终生成质量优于竞品。本轮已补 `scripts/run_real_eval.sh`、`real_full` / `real_no_rag` / `real_no_reflexion` 真实评测入口、RAG preflight 和报告 `Judge 来源` 标记；timicc OpenAI-compatible `/responses` 中转已通过 `scripts/check_llm.sh`。进一步收敛后，`real_full` 小样本已进入真实 Qdrant RAG 链路，`overall=0.6900`，高于 `real_no_rag overall=0.5467` 和 `single_agent_baseline overall=0.2000`。但这仍是 1 条 case 的阶段证据，真实 RAG/Reflexion 的比赛级结论仍需扩大样本和人工抽检。

### 6.4 已修复的并发坏味道

`KeywordIndex.get()` 曾在 async 函数内用 `threading.Lock` 包裹 `await _build_from_qdrant()`。`real_full` 并发生成 doc/mindmap/reading 时，第二个任务同步抢锁会阻塞事件循环，导致第一个任务也无法恢复，表现为真实评测外层超时。本轮已改为 `asyncio.Lock` 并新增并发回归测试。后续审查 async 代码时，要重点排查同步锁、同步 I/O、大模型加载是否进入事件循环关键路径。

### 6.5 测试目录已经过大

`tests/unit/` 文件数量较多，超过项目“每层尽量不超过 8 个文件”的约束。建议后续拆为：

```text
tests/unit/api/
tests/unit/common/
tests/unit/data_engineering/
tests/unit/eval/
tests/unit/executor/
tests/unit/memory/
tests/unit/orchestration/
tests/unit/rag/
tests/unit/skills/
```

### 6.6 个别源码文件偏长

当前有少量 Python 文件接近或超过 300 行约束，应在后续功能稳定后分拆，例如：

- `src/reflexlearn/skills/path_plan.py`
- `src/reflexlearn/data_engineering/ingest.py`
- `src/reflexlearn/data_engineering/graph_build.py`
- `src/reflexlearn/orchestration/graph.py`

## 7. 下一阶段建议

推荐下一阶段顺序：

1. M5 真实评测扩大样本：把 `rag_required` / `reflexion_required` 从 1 条 case 扩到多条，并补人工抽检，形成可信报告。
2. reranker 增强验证：资源足够时显式 `ENABLE_RERANK=true`，评估 bge-reranker 精排是否带来稳定收益。
3. Grafana 面板人工检查：本轮 `/metrics` 与 observe 容器已可用，下一步用浏览器确认 `http://localhost:13001` dashboard 自动加载并有数据。
4. MCP 客户端联调：SDK 与启动脚本已通，下一步接真实 MCP client 验证 `list_tools` 与工具调用。
5. P0 对象级权限、审计、AI Safety Gateway 与上传隔离：知识资源、视频任务、上传文档全部绑定用户/租户并校验归属。

## 8. 审查结论

当前项目已经完成“能跑、能演示、能继续迭代”的阶段性目标。多智能体编排、RAG、记忆、数据工程、前端演示、安全底座都有实代码和测试支撑。

但项目不能按“生产可上线”宣传。下一阶段必须优先补齐对象级权限、审计、AI Safety Gateway 和上传隔离，否则多用户环境下仍存在安全和合规风险。
