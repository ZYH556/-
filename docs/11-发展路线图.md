**排序决策（2026-06-08）**：`docs/10 §7` 原建议「安全优先」。本轮经用户拍板改为「**演示拿分优先**」——阶段二提为当前主线，阶段一顺延。

**安全护栏（硬约束）**：阶段二顺延安全闭环期间，**不得引入新的多用户暴露面**——不新增可被越权访问的对象级数据通路、不放开当前 P0 安全开关、演示环境保持单租户或可信内网。这样把「安全顺延」的风险锁死在「现有暴露面」内，不让它扩大。

**砍单顺序**：时间不够时从阶段四往上砍（LoRA → 爬虫 → 实时增量链路 → 多模态降级图文）。**P0（阶段一）永不砍。**

## 3. 阶段一 · P0 安全合规闭环

> 目标：从「最小安全底座（55%）」补到「多用户可上线」。docs/10 §7 列为最高优先级，是当前最大合规风险。

| 子项 | 任务 | DoD（验收标准） |
| --- | --- | --- |
| 1.1 对象级权限 | 知识资源 / 视频任务 / 上传文档全部绑 `user_id+tenant_id`，读写校验归属 | 跨用户/跨租户访问他人资源返回 403；单测覆盖越权路径 |
| 1.2 AI Safety Gateway | 统一输入/输出审核、越权提示识别、敏感信息过滤、审计记录 | 恶意提示/敏感输出被拦并记审计；降级矩阵齐全 |
| 1.3 上传隔离升级 | 隔离区 → 扫描占位 → 内容审核 → 签名 URL → 防盗链 | 上传文件先入隔离区、扫描通过才转正；下载走签名 URL |
| 1.4 会话安全 | DB 用户体系 + HttpOnly Cookie + CSRF + 登录限流 + 审计日志 | 替换当前 sessionStorage Bearer MVP；登录暴力破解被限流 |

**风险**：对象级权限会改动 chat/knowledge/video 三组接口与 DB schema，须保「零回归 + 降级铁律」；AI Safety Gateway 无 LLM key 时要规则降级，不可阻断主链路。

### 4.2 M5 真实评测结论 ✅/🚧 real_full 小样本已通，待扩大样本

- **本轮状态（2026-06-08）**：已新增 `--real` 真实评测模式、`real_full` / `real_no_rag` / `real_no_reflexion` 策略、`scripts/run_real_eval.sh`、RAG preflight，并在单策略/对比 Markdown 报告中标注 `Judge 来源`。普通 `run_eval.sh --compare` 默认仍只跑 smoke/受控策略，不会误触真实 RAG 模型加载。
- **LLM 接入**：`LLMGateway` 已支持 OpenAI-compatible 中转站，填 `OPENAI_COMPAT_API_KEY` / `OPENAI_COMPAT_BASE_URL` / `OPENAI_COMPAT_MODEL` 后优先使用；`OPENAI_COMPAT_WIRE_API=responses` 已实测适配 timicc `/responses`。
- **已验证**：`bash scripts/check_llm.sh` 通过，`model_used=openai/gpt-5.5` 且 `json_parse=ok`；`bash scripts/run_eval.sh --compare --tags ablation,rag_required --strategies controlled_rag,single_agent_baseline --max-cases 1 --timeout 45` 通过，`controlled_rag overall=0.4967`，`single_agent_baseline overall=0.1800`，输出 `Judge 来源=LLM 或混合`。
- **新增验证**：`bash scripts/run_real_eval.sh --tags ablation,rag_required --strategies real_full,real_no_rag,single_agent_baseline --max-cases 1 --timeout 180` 通过；`real_full overall=0.6900`、coverage 100.0%，`real_no_rag overall=0.5467`、coverage 100.0%，`single_agent_baseline overall=0.2000`、coverage 33.3%。
- **关键修复**：`KeywordIndex.get()` 从同步锁包裹 await 改为 `asyncio.Lock`，修复 `real_full` 并发资源生成时的事件循环死锁；`run_real_eval.sh` 默认 `ENABLE_RERANK=false`、`REAL_EVAL_TIMEOUT=180`，避免默认加载第二个重模型。
- **剩余任务**：扩大 `rag_required` 样本；放开真实 Reflexion 跑 `reflexion_required` 切片；抽样人工复核 LLM judge；资源足够时显式 `ENABLE_RERANK=true` 评估 reranker 增强。
- **DoD**：`docs/08` 升级出「真实 LLM judge + 真实 RAG/Reflexion 消融 + 人工抽检」的可信报告；消融对比表能体现单/多 Agent、有/无 RAG、有/无 Reflexion 的真实差距。
- **诚实限制**：当前原生 DeepSeek/Qwen/Anthropic key 为空，OpenAI-compatible timicc 已配置并通过 `check_llm.sh`；`real_full` 结论是 1 条 case 的阶段证据，不能替代扩大样本和人工抽检。
