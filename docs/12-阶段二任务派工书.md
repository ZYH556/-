## 2. 全局开发铁律（所有卡必须遵守，违反即打回）

这些是项目宪法，每张卡都默认适用，卡内不再重复，只在有针对性补充时点名。

1. **降级铁律（最高优先级）**：**绝不假装成功、绝不报错中断。** 外部依赖（LLM/DB/模型/broker）不可用时走本地 fallback，不抛异常打断主链路。**每个新功能必须配「降级矩阵」**（列出每种依赖失效时的行为）。
2. **依赖注入，不函数内自取**：需要 pg/redis/neo4j/qdrant/外部 client 时，**作为参数注入**，不在函数体内 `get_pg_pool()`/`get_redis()` 自取。原因：`tests/conftest.py` 的 hermetic 守卫**不拦** `get_redis`/`get_pg_pool`/`get_neo4j`，函数内自取会在单测里触发真实连接卡死。范式参考 `data_engineering/ingest.py`、`memory/manager.py`。
3. **hermetic 单测**：conftest 拦 `_get_model`/`_get_reranker`/`get_qdrant`；**不拦** redis/pg/neo4j。新功能必配单测 + **降级路径测试**；涉及 redis/pg/neo4j 的，在封装层 mock 或注入假对象，**绝不让被测函数真连**。
4. **配置范式**：新配置加到 `common/config.py` 的 `Settings`，用 `enable_xxx: bool` 开关 + 合理默认值 + 行内中文注释；读配置走 `get_settings()`（带 `@lru_cache`，测试切环境后需清缓存）。
5. **文件/目录约束**：单个源码文件 **≤300 行**；每个目录 **≤8 个文件**（超了先分子目录）。
6. **Run & Debug 走脚本 + 写日志**：所有启动/调试/检查操作封装进 `scripts/*.sh`，并 `tee -a "$LOG_DIR/*.log"` 写 `logs/`。新增脚本复用 `scripts/_lib.sh`（`PROJECT_ROOT`/`LOG_DIR`/`ensure_logs`/`.venv` 定位）。
7. **零回归**：读 state 新字段用 `.get()`；不改既有节点/skill 契约（节点返回 dict 合并进 state；skill 返回 `SkillResult(ok,data)`；`assemble_node` 不动）。
8. **环境陷阱**（踩过的坑）：后台 bash 命令 cwd 不在项目根 → 命令开头 `cd /d/2026/multagent &&`；带 RAG 的后端与单测**不要同时**加载 bge 模型（双份 2GB 模型换页卡死）；curl 传中文 body 用 `--data-binary @body.json`（UTF-8 文件）。

### 3.2 设计决策（已替你拍板，照做即可）

- **用 `prometheus-client` 拉取式指标，不引入 OpenTelemetry 重栈**。理由：本项目是单体 FastAPI + LangGraph，Prometheus 指标 + 结构化日志里的 `trace_id` 足以支撑"技术质量"演示；OTel 分布式 trace 对单体收益低、接入成本高。**LangSmith trace 列为可选增强**（需 key，见 3.8），不在本卡 DoD。
- **全局唯一 registry，避免重复注册**：prometheus-client 默认 `REGISTRY` 是全局单例，模块被多次 import 或测试重载会抛 `Duplicated timeseries`。**所有指标在 `observability/metrics.py` 顶层定义一次**，其它模块只 import 使用，不重复 `Counter(...)`。
- **埋点不得违反降级铁律**：埋点代码本身要 try/except 兜底，**指标采集失败绝不能影响业务**（监控挂了不能拖垮服务）。
- **指标默认开启、可配置关闭采集**：`common/config.py` 已新增 `enable_metrics: bool = True`。关闭时保留 `/metrics`，但业务路径不新增指标样本；`/metrics` 保持匿名访问（Prometheus 抓取无 token），不挂在受保护路由下。
