## 2. 产品形态：三层学习平台

### 2.1 三层结构

| 层 | 内容 | 性质 | 落点（现有/新增） |
|---|---|---|---|
| **① 公共内容生态**（卖点/获客） | 按**赛道**组织的学习馆（求职/考证/职业培训/技能…），每赛道含公共课程、知识库、资源 | `visibility=public`，全员共享、个人不可改 | 新增赛道/课程 PG 表 + 公共知识灌入 `data_engineering/`；初期 1 个示范赛道（AI/编程，复用现有 ML 知识库） |
| **② AI 个性化引擎**（核心亮点=ReflexLearn 本体） | 聊天发起 → 多智能体协作 → 构建「属于你的智能体」 | 引擎 | 现有 `orchestration/` 主链路 + 自进化飞轮（§3） |
| **③ 个人学习数据**（黏性/壁垒/飞轮燃料） | 学习计划、路径与进度、个人知识库、**错题本**、资源库、成长档案、个人画像/记忆 | `user`+`tenant` 私有隔离 | 新增 `learning/` 模块 + PG 表 + 个人空间 API |

### 2.2 核心机制：公共资源 × 个人智能体 = 千人千面

公共内容**不被复制、不被修改**；个人智能体作为「个性化重组器」：检索（公共库 + 个人私有库的混合 RAG）→ 按画像/进度/错题重组 → 生成你的专属计划与资源。**写入只落个人层，绝不污染公共层**。这正是既有 `RAG（公共+私有检索）+ path_plan（个性化路径）+ 记忆（懂你）` 的产品化呈现。

### 2.3 信息架构（前端）

```
公共门户（未登录可浏览，登录后个性化）
 ├─ 首页 / 赛道馆（求职·考证·职业培训…）
 ├─ 赛道详情：公共课程 / 公共知识库 / 推荐学习路线
 └─ 登录 / 注册

个人工作台（登录后）
 ├─ 我的学习空间列表（按学习目标组织：每个空间=知识库+资源库+路径+进度+画像）
 ├─ 对话工作区（聊天发起 → Agent 协作时间线 → 资源流式生成）
 ├─ 学习计划与路径（可视化、可跟踪、随进度自调整）
 ├─ 资源库（已生成的 doc/mindmap/code/quiz/reading/video 管理）
 ├─ 个人知识库（上传/管理私有资料）
 ├─ 错题本（错题 → 归因 → 复习 → 针对性资源）🔥 飞轮燃料口
 ├─ 成长档案（画像可视化、薄弱点、自进化轨迹「它如何越来越懂你」）
 └─ 系统状态 / 设置
```

## 3. 内核：自进化飞轮 Self-Evolution Flywheel

### 3.1 双回路架构

```
                       用户交互 / 学习目标
                              │
                   ┌──────────▼───────────┐
                   │  多智能体协作（生成资源）  │ ──► 资源 + 协作轨迹
                   └──────────┬───────────┘
              快回路（在线·秒级·免训练）       慢回路（离线·天级·需GPU）
        ┌───────────────┴────────────┐         │
        ▼                            ▼          ▼
  元认知自我改进                 记忆系统进化     协作轨迹积累到阈值
 （即时纠正 plan/prompt/tool）  （经验沉淀→召回）      │
        │                            │          ▼
        └──────────► 当轮/跨会话变聪明 ◄┘     LoRA 微调（经验固化进权重）
                                                  │
                                                  ▼
                                            更强基座 → 更好协作 →（飞轮闭环）
```

### 3.2 四方向：现状 → 跃迁 + 关键设计

| 方向 | 现状（已有） | 升级跃迁（创新点） | 落点 | 回路 |
|---|---|---|---|---|
| **记忆系统进化** | 三级记忆（trim/递归摘要/Reflexion 语义召回）+ Redis 多轮 + profile 跨会话 | 记忆巩固（重要经验晋级长期）/ 主动遗忘（时间衰减+低价值淘汰）/ 知识图谱自生长（交互自动抽概念入图，复用 M4-B 抽取）/ 记忆复用度量 | `memory/` | 快 |
| **元认知自我改进** | Reflexion 写反思 + critic/debate 节点 | 显式**元认知节点**：执行后自评 plan/prompt/tool 哪里可改 → self-refine 重试；失败 case 自动 prompt 优化并回验；自动工具选择 | `orchestration/nodes/metacognition.py`(新) + `harness.py` + `skills/` | 快 |
| **多智能体协作深化** | central/pipeline/debate 三模式 + judge | 角色分化（专长 Agent）/ 动态编排（按任务自适应选协作模式）/ Agent 间博弈与共识协议 / **协作轨迹记录管线**（为 LoRA 产数据） | `orchestration/graph.py` + `nodes/` + 新 `trace/` 轨迹落库 | 慢（产料）|
| **自进化闭环 LoRA** | M5-E 仅规划 | 轨迹 → 训练数据管线 → AutoDL LoRA 微调 → eval 证收益 → vLLM 热加载微调权重 | 新 `finetune/` + `eval/` + `llm_gateway/`（权重路由） | 慢 |

### 3.3 飞轮如何对用户「可见」

抽象的「自进化」必须变成用户**看得见、用得爽**的功能，否则只是 PPT 词：

- **错题本**（🔥 燃料口）：做错题 → 元认知分析「为什么错」→ 记忆沉淀薄弱点 → 学习计划自动调整 → 协作生成针对性资源 →（长期）喂给 LoRA。**一个功能串起全部四个方向**。
- **成长档案**：可视化「系统如何越来越懂你」——画像演化、薄弱点收敛曲线、自我改进前后对比。
- **越用越准的推荐**：同一公共内容，随使用次数增加，路径/资源推荐越贴合该用户。

### 3.4 降级铁律延续（项目宪法，每个内核必配）

沿用全项目铁律——**绝不假装成功、绝不报错中断**。每个新内核必须配降级矩阵：记忆服务挂→退当前行为；元认知 LLM 不可用→跳过自评走原结果；协作轨迹落库失败→不阻断主链路；LoRA 权重不可用→回退基座模型。依赖一律**注入不自取**（conftest 不拦 redis/pg/neo4j），新功能必配单测 + 降级路径测试。

## 4. 门面：灵动玻璃设计系统

### 4.1 风格定义

对标 Arc / Apple 液态玻璃：毛玻璃景深（backdrop-blur 多层）、半透明分层、流体动效、圆润大圆角、柔和光晕/渐变、暗色优先。**刻意摆脱「普通 AI 前端」的廉价同质感**——复写大牌交互组件而非堆砌默认 UI 库。

### 4.2 技术选型（在现有栈上加，不换栈）

| 用途 | 选型 | 说明 |
|---|---|---|
| 设计 token | Tailwind v4 CSS-first（`@theme`） | 玻璃材质/景深/光晕/动效曲线统一为 token |
| 玻璃组件库 | 自建 `frontend/components/glass/` | 复写卡片/面板/侧栏/弹层/按钮，含 backdrop-blur 分层规范 |
| 动效 | Framer Motion（或 CSS view transitions） | 流体过渡、微交互、列表 reorder、页面切换 |
| 既有渲染 | 保留 react-markdown/KaTeX/Markmap/Shiki | 资源卡片渲染不动，只换外壳风格 |

### 4.3 渐进改造策略（保护已验证链路）

1. **先建设计系统**（token + 玻璃组件库 + 动效规范），不动业务逻辑。
2. **新建工作台路由**（公共门户 + 个人工作台 IA），与现有单页演示台并存。
3. **逐模块迁移**：已活体验证的 SSE 流式 / 知识上传 / 视频轮询**逻辑不动**，只换玻璃外壳。
4. 性能守门：玻璃多层 blur 有 GPU 成本，移动端/低端机降级为半透明纯色；首屏预算守住（现 467kB 基线，新增按需 code-split）。

## 5. 底座：多租户安全 + 部署（产品内核，非合规负担）

> **正名**：`docs/11` 阶段一的「P0 安全」在产品平台语境下**不是顺延的累赘，而是产品定义的必要组成**——每个用户的学习空间/知识库/记忆/错题必须物理隔离，否则「个性化学伴」根本不成立。它被提到与内核同级。

| 子项 | 内容 | 落点 |
|---|---|---|
| 对象级权限归属 | 学习空间/资源/视频/上传/错题全绑 `user_id+tenant_id`，读写校验归属，越权 403 | `api/deps` + 各 route + PG schema |
| 账户体系 | DB 用户体系 + HttpOnly Cookie + CSRF + 登录限流 + 审计（替换当前 sessionStorage Bearer MVP） | `common/auth` + 新 `accounts/` |
| AI Safety Gateway | 输入/输出审核、越权提示识别、敏感过滤、审计；无 LLM key 规则降级不阻断 | 新 `safety/` |
| 上传安全 | 隔离区 → 扫描占位 → 内容审核 → 签名 URL → 防盗链 | `api/upload_guard` 升级 |
| 部署 | docker-compose 全栈上服务器 + 监控（已有 Grafana）+ 备份 | `deploy/` + `docker-compose.yml` |

## 7. 验收与度量口径

- **统一接 M5 eval**：每个内核升级都要在 `eval/` 的 `ablation`/`rag_required`/`reflexion_required` 切片上跑前后对比，用 `overall`/`correctness`/`resource_coverage` 量化 Δ。现成对照基线：`real_full 0.6900 / real_no_rag 0.5467 / single 0.2000`。
- **自我改进可量化**：元认知升级要证明「self-refine 前后」同 case 分数提升；记忆升级要证明「跨会话复用」命中率提升；LoRA 要证明「微调 vs 基座」收益。
- **可观测覆盖**：新链路（记忆复用、元认知节点、协作轨迹、错题本、LoRA 推理）补 Prometheus 指标，进 Grafana 面板。记忆复用指标已先行落地。
- **诚实红线**：任一真实环境跑不通（GPU 不足/数据量不够/收益为负），**如实标注限制，不得粉饰**（项目一贯纪律）。

## 9. 风险与诚实边界

| 风险 | 影响 | 预案 |
|---|---|---|
| LoRA 收益不确定 / 数据量不足 | 中 | 作为慢回路、可砍；先把轨迹数据管线跑厚，eval 证明再上；AutoDL 按需租 |
| 玻璃多层 blur 性能成本 | 中 | 低端设备降级半透明纯色；首屏预算守门 + code-split |
| 安全工作量大、改动面广 | 中高 | 对象级隔离最先做（产品地基）；改 chat/knowledge/video 三接口须零回归+降级铁律 |
| 长期演进战线长、易失焦 | 中 | 每波次有可演示版本；波次内子项接口先行、可独立派工；ROI 高的快回路先行 |
| 公共/个人数据边界泄漏 | 高 | 写入只落个人层；RAG 读混合但 ACL 过滤；越权用例必配单测 |
