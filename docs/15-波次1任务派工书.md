## 2. 卡 W1-A · 记忆系统进化

> **一句话**：把记忆从「存取」升级为「新陈代谢」——重要经验巩固晋级、低价值经验时间衰减淘汰、交互自动让知识图谱生长、复用度量闭环。对应 `docs/14` §3.2 记忆方向。

### 2.1 现状（已核实）
- `memory/manager.py:9` `MemoryManager`——无状态门面，依赖注入 `qdrant/pg_pool`；`recall(task_type, query, acl)`→ 转发 `recall_reflections`；`recall_memory_node(state)` 在主链路 `recall` 节点调用（`graph.py:61` 闭包注入 `_memory_manager`）。
- `memory/reflexion.py`——经验记忆核心：
  - `write_reflection(*, pg_pool, qdrant, reflection: Reflection, user_id) -> bool`（`:25`）：PG 持久化 + Qdrant 语义写入；**当前 Qdrant payload（`:230-237`）只有** `user_id/task_type/failure_type/cause/fix_strategy/success`——**没有 `created_at`、没有 `hit_count`**（这是巩固/遗忘的硬前置）。
  - `recall_reflections(*, qdrant, task_type, query, acl, limit=3) -> list[dict]`（`:43`）：语义召回→scroll 降级；**已接 `observe_memory_recall(mode, status, result_count)`**（复用度量指标已落地，见 `:52/64/84`）。
  - 集合常量 `EXPERIENCE_COLLECTION = "experience_memory"`（`:22`）。
- `data_engineering/graph_build.py`（M4-B，已存在）：`extract_concepts(...)` + `merge_into_neo4j(...)` + `build_graph(...)`——**图谱自生长可直接复用，不重写抽取**。schema 对齐读侧 ACL（概念须带 `tenant_id+visibility`，见 memory「图谱写读 ACL 命门」）。
- `config.py` 记忆开关现状（`:24-39`）：`enable_reflexion / enable_multi_turn / session_ttl / profile_ttl / enable_promote / summary_*`。

### 2.2 设计决策（已拍板）
- **先补 payload 字段，再做巩固/遗忘**：`write_reflection` 的 Qdrant payload 增加 `created_at`（ISO 字符串，由调用方传入时间戳——脚本/节点侧 stamp，**不在纯函数里取 `Date.now`**）和 `hit_count: 0`。老 point 无字段→读时 `.get(k, 默认)` 向后兼容。
- **巩固 = 召回即回写 hit_count**：`recall_reflections` 命中某 point 后异步 `+1` 其 `hit_count`（best-effort，失败不影响召回返回）。`hit_count` 高 = 高价值 = 遗忘时豁免。
- **遗忘 = 独立离线任务，不进主链路**：新 `scripts/run_forget.py`（仿 `scripts/kafka_consumer.py` 入口范式）扫描 `experience_memory`，按「`created_at` 超 `memory_ttl_days` 且 `hit_count` 低于阈值」删除。**绝不在 recall/请求链路里删**（避免请求副作用 + 性能）。
- **图谱自生长 = 会话结束触发，复用 M4-B**：在 `run_session` PERSIST 段（`graph.py:227` 之后，与 `enable_promote` 并列），`enable_graph_autogrow=true` 时把本轮 assistant 摘要/资源文本喂 `graph_build.extract_concepts` → `merge_into_neo4j`（带 `tenant_id+visibility=public`）。**全程包在已有的 `try/except: pass` 里**，失败不影响响应。
- **复用度量已落地**：本卡只需把 `hit_count` 巩固与已有 `observe_memory_recall` 联动，不重复造指标。

## 3. 卡 W1-B · 元认知自我改进

> **一句话**：现有 `critic` 是「失败后才反思」；本卡加**主动元认知**——执行后即使没失败也自评 plan/prompt/tool 哪里可改，做有限次 self-refine；并让失败 case 自动改写指令重试。对应 `docs/14` §3.2 元认知方向。这是波次 1 里**最能在 eval 上出 Δ** 的一卡。

### 3.1 现状（已核实）
- 主链路（`graph.py:111-126`）：`START→profile→recall→planner→[dispatch_route: generate_resource|pipeline]→gate→[gate_route: critic|debate|assemble]→...→assemble→path_plan→END`；`critic→planner`（replan 回环）。
- `nodes/critic.py:20` `critic_node(state)`：只处理 `completed` 里 `status=="failed"` 的任务 → LLM 反思成 `Reflection` → 持久化 → `replan_count+1`、`iteration+1`。**没失败就直接返回**（`:22`）。这是 self-refine 要扩展的范式。
- `harness.py:10` `harness_guard(node_fn)`：`iteration>=max_iterations` 或 `token_used>=token_budget` 即 halt。**self-refine 次数闸门应复用这套**（或加 `replan_count<max_replan` 判断，`config.py:109 max_replan=3`）。
- `LLMGateway.complete(messages, *, task_type="generation", schema=None, temperature=0.7) -> Completion`（`gateway.py:30`）：无 key 抛 `RuntimeError("llm_no_api_key")`，传 `schema`（pydantic）走 JSON 结构化输出。
- `Skill` 接口（`skills/base.py`）：`SkillContext(user_id, acl, task_id, trace_id)` / `SkillResult(ok, data, error_type, duration_ms, cached)` / `async run(inp, ctx)`。
- `AgentState`（`state.py:20`）：`completed: Annotated[list, add]`（reducer）、`reflections`、`replan_count`、`iteration`、`learner_profile` 等。

### 3.2 设计决策（已拍板）
- **新增 `metacognition` 节点，不改 critic 语义**：新 `nodes/metacognition.py:metacognition_node(state)`——对 `completed` 里 `status=="passed"` 的资源做**质量自评**（LLM 打分 + 指出可改点），低于阈值的标记 `needs_refine` 并构造改进 spec。
- **零回归优先：默认关，开启才插入回路**。`enable_metacognition=False`（默认）时主链路与现在**逐字节等价**。开启时在 `gate_route` 增加分支或在 `gate→assemble` 前插入 `metacognition`，self-refine 把 `needs_refine` 任务重置为 `pending` 回 `generate_resource`，受 `max_self_refine` 闸门限制。
- **self-refine 闸门复用 harness 范式**：新增 `self_refine_count` 到 state，`>= max_self_refine` 即停（仿 `replan_count<max_replan`），绝不无限循环。
- **自动 prompt 优化 = 元认知产出 fix_spec 注入下一轮生成**：失败/低分任务，元认知输出「指令应如何改」写进 `task["spec"]["refine_hint"]`，`generate_resource` 读 hint 增强 prompt。复用现有 generator，不新建生成器。
- **自动工具选择留接口、本卡做轻量版**：元认知可建议「该任务更适合哪个 skill / 协作模式」，写入 state 供 planner 下一轮参考；**完整动态编排留波次 2 协作深化**（本卡不展开，避免和波次 2 重叠）。

|---|---|
| `orchestration/nodes/metacognition.py`（**新**，≤300 行） | `metacognition_node(state)`：自评 passed 资源、产出 refine_hint、标 needs_refine；LLM 无 key→跳过（降级铁律） |
