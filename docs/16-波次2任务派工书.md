## 1. W2-0 · 真实活体验证前置

**目标**：把波次 1 的诚实限制转成可复跑脚本，不再只停留在口头限制。

**验收项**：
- 真实 Qdrant 遗忘删除活体：写入过期低命中经验点，执行遗忘，确认点被删。
- 真实 Neo4j 图谱自生长写入活体：复用 `memory/graph_autogrow.py`，确认概念节点可查。
- 带 key 元认知 self-refine 对比：`metacognition_on` trace 进入节点，记录 on/off 分数。
- 跨用户 403 用例：至少覆盖视频作业或错题对象。
- 协作轨迹可查询：触发一次 chat/session 后能查到节点事件。

**降级矩阵**：
| 依赖 | 行为 |
|---|---|
| Docker/Qdrant/Neo4j 不可用 | 脚本输出 blocked 原因，不伪造成功 |
| LLM key 不可用 | 元认知真实收益标记为 skipped，保留接线 smoke |
| PG 不可用 | 协作轨迹走内存降级，主链路不中断 |

## 7. W2-H · 元认知性能治理

**目标**：`metacognition_real_on` 稳定进入元认知节点并完成一次 self-refine，同时把总耗时压到 45s 可评测区间内，避免 Judge 超时规则降级。

**涉及文件**：
- `src/reflexlearn/common/config.py`：新增 `eval_judge_max_resources`，保留元认知限审、超时、截断配置。
- `src/reflexlearn/eval/strategies.py`：`metacognition_real_off/on` 限定 1 个 Judge 样本；`metacognition_real_on` 单独放宽元认知 LLM 超时到 20s。
- `src/reflexlearn/eval/runner.py`：生成资源覆盖率仍按全部资源统计，LLM Judge 只评 doc 优先的前 N 个资源。
- `src/reflexlearn/orchestration/nodes/metacognition.py`：短 prompt + 离线占位低分约束 + 超时/坏 JSON 可解释 degraded review。
- `src/reflexlearn/skills/offline.py`：离线 doc 二次生成写入 `previous_issues` 的“元认知修复建议”。
- `scripts/check_metacognition_perf.sh`：W2-H 真实性能验收入口。

**DoD**：
- `metacognition_real_on` trace 包含 `metacognition`。
- `self_refine_count=1`，且最多一次。
- `metacognition_real_on latency_ms <= 45000`。
- Judge reasoning 不是 `rule:` 规则降级。
- `metacognition_real_on` overall 不低于 off 超过 0.03。

**已验收结果**：
- `bash scripts/check_metacognition_perf.sh` 通过：最新验收轮 `latency_ms=38982`，`off=0.18`，`on=0.55`，trace 含 `metacognition` 与 `self_refine_count=1`。
- `bash scripts/test_unit.sh`：397 passed, 2 warnings。
- `bash scripts/build_frontend.sh`、`bash scripts/check_wave2_api.sh 8000`、`bash scripts/check_frontend_ia.sh`、`bash scripts/check_frontend_glass.sh`、`bash -n scripts/*.sh`、`git diff --check` 均通过。

**诚实限制**：不能宣称元认知稳定提升学习效果；当前只是 1 个 `reflexion_required` case 的真实性能链路证据，后续仍需扩大样本、稳定 LLM Judge 和人工抽检。
