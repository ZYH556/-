# 20 · 首页功能叙事与后期页面内容梳理

## 1. 本次首页调整目标

首页要表达的不是“后台工作台”，而是一套面向学习者的 AI 个性化学习系统。首屏继续保留夜空视频背景和居中标题，向下滚动后用内容区说明 ReflexLearn 如何把一次学习目标转化为长期成长闭环。

当前首页叙事主线：

1. 用户提出学习目标。
2. 系统建立学习画像。
3. AI 规划学习路径并匹配导师。
4. 多 Agent 生成学习资源。
5. 课程学习、讨论、AI 辅导和错题复盘持续产生反馈。
6. 差距分析和成长档案推动下一轮学习计划。

## 2. 截图功能内容归纳

### 2.1 精品课程与课程播放

截图展示了精品课程入口、学科分类、课程推荐、课程播放页、章节目录和课程讨论区。课程不是孤立视频，而是学习系统中的基础内容源。用户进入课程后，可以看到章节顺序、学习时长、课程评分和讨论内容。

首页承接文案应强调：

- 覆盖多学科课程分类和学习路线；
- 支持从课程播放进入章节化学习；
- 讨论区问题可由 AI 生成回复，帮助学习者及时获得解释。

### 2.2 AI 智能辅导

截图中的 AI 辅导页基于学习画像和薄弱知识点，向学生提出诊断问题，并给出后续学习计划入口。右侧展示学习画像、薄弱环节、错题总结和快捷工具。

首页承接文案应强调：

- AI 不是泛泛聊天，而是围绕画像和错题提问；
- 系统能识别薄弱知识点；
- 辅导结果可以连接到练习、资源推荐和面试辅导。

### 2.3 导师智能匹配

截图展示了目标学生、课程方向、匹配人数、匹配维度、推荐导师卡片和匹配度。AI 智能导师与真人导师可以共同构成辅导网络。

首页承接文案应强调：

- 根据学习风格、知识短板、目标方向和课程需求匹配导师；
- 展示匹配度和推荐理由；
- 支持 AI 导师全天候响应，真人导师提供更深层指导。

### 2.4 AI 学习路径规划

截图展示了目标画像、学习目标、学习周期、AI 生成路径、章节进度、难度分布、预计完成时间和推荐理由。路径页将抽象目标拆成章节、重点、关联知识点和阶段进度。

首页承接文案应强调：

- 从目标到章节的可执行路线；
- 每一章包含重点、难度、预计耗时和当前学习位置；
- 路径会结合学习画像和后续反馈动态调整。

### 2.5 学习画像与成长档案

截图展示了个人画像、雷达图、学习活跃度、能力评估、学习天数、考试次数、面试次数、正确率、错题分析和最近错题。画像不是静态资料，而是系统调整路径和资源的依据。

首页承接文案应强调：

- 画像由学习行为、错题、能力维度和目标方向共同构成；
- 成长档案用数据说明当前状态；
- 错题和能力变化会反向影响下一轮计划。

### 2.6 差距分析与行业画像

截图展示了 AI 差距分析报告、行业标准能力对比、匹配度、优势能力、待提升项、需重点关注项和个性化建议。行业能力画像帮助学习者理解“还差什么”。

首页承接文案应强调：

- 对照行业能力标准，而不是只看学习完成度；
- 明确优势、短板和重点关注项；
- 将差距转化为后续资源、练习和项目训练建议。

### 2.7 多 Agent 资源生成

截图展示了资源生成平台、生成类型、Agent 流水线和资源库。系统可生成课程文档、思维导图、练习题库、拓展阅读、视频脚本和代码实操。

首页承接文案应强调：

- 多 Agent 分工生成不同形态资源；
- 资源围绕当前目标和路径生成；
- 生成内容进入资源库，成为可复用的学习资产。

## 3. 首页内容映射

| 首页区块 | 对应产品能力 | 推荐表达重点 |
| --- | --- | --- |
| 首屏 Hero | 产品总定位 | 会记住、会规划、会生成资源、会反馈的学习系统 |
| 窄列叙事 | 学习目标到成长闭环 | 先建立气质和信任，不堆页面截图 |
| 能力横向卡片 | 画像、路径、导师、差距、资源、错题、课程、AI 互动 | 用短句说明“用户能得到什么” |
| 数据带 | 资源形态、检索召回、画像维度、智能辅导响应 | 保持真实可信，避免浮夸数字 |
| Agent Workflow | 画像 Agent、路径 Agent、资源 Agent、评估 Agent | 解释 AI 智能体如何协作，而不是只说用了 AI |
| Product Scenes | 后期页面功能串联 | 按“画像 - 路线 - 资源辅导 - 成长反馈”组织 |
| Growth Loop | 学习路径、资源库、错题分析、差距报告 | 用不同底色增强层次，避免全页米白单调 |
| Footer | 品牌和核心入口 | 保留深色收尾，提高文字清晰度 |

## 4. 后续图片填充建议

后续补图时建议不要把所有后台截图平铺到首页。更适合的方式是：

1. 首屏仍保留夜空视频，不替换背景。
2. 能力卡片区可加入抽象化的产品局部截图，例如路径进度、画像雷达图、资源生成流水线。
3. Product Scenes 区块可放 2 到 4 张精选页面截图，优先选择学习路径、资源生成、差距报告、课程讨论 AI 回复。
4. 图片需要统一裁切比例和边角，不直接使用带浏览器地址栏、红色标注或后台侧边栏过重的原图。
5. 如果要展示后台级页面，建议先做二次设计稿，保留核心内容，弱化管理系统痕迹。

## 5. 仍需确认的设计点

1. 首页是否继续使用英文导航 `Log in`，还是改为中文“登录”。
2. 首屏标题是否保留当前“记住、进化”的表达。
3. 下方内容是否继续偏品牌叙事，还是更快进入产品截图展示。
4. 数据带数字是否继续使用当前真实工程数据，还是替换为课程、资源、画像维度等产品数据。
