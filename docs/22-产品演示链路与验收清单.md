# 22 · 产品演示链路与验收清单

更新时间：2026-06-12  
文档性质：正式验收文档  
适用范围：登录后学习系统、产品演示、功能验收、答辩走查

---

## 1. 演示目标

本演示不把 ReflexLearn 展示成后台系统，而是展示成一个 AI 个性化学习系统：

> 用户登录后，系统直接告诉他今天该学什么、为什么这么学、从哪里继续，并把学习空间、资源、AI 导师、错题复盘和成长档案串成闭环。

演示必须让观众看到三件事：

1. 系统能理解学习者当前目标和薄弱点。
2. 系统能组织资源、路径和错题，给出下一步行动。
3. AI 导师不是普通聊天，而是学习画像、路径和资源生成的入口。

## 3. 推荐演示主线

### 3.1 登录进入 Today

入口：`/today`

讲解重点：

- 登录后不是进入后台列表，而是进入“今日学习”。
- 页面直接回答：今天建议学什么、为什么、从哪里开始。
- Today 读取 `GET /api/today`，接口失败时显示离线学习建议。

验收点：

- 页面第一屏能看到当前目标、今日主任务、路径预览、推荐资源、AI 导师提示、复习提醒。
- 主任务按钮有明确动作，不是泛泛菜单。
- 文案不出现“工作台”“本地草图”“后续接入”等内部词。

### 3.2 从 Today 进入学习空间详情

入口：Today 主任务按钮 -> `/spaces/[id]`

讲解重点：

- 学习空间是一个学习目标的容器，不是普通文件夹。
- 它聚合路径、资源、进度和下一步建议。

验收点：

- 顶部有学习目标标题、状态和进度。
- 主区有学习路径节点。
- 右侧或辅助区域有下一步行动建议。
- 下方有该空间沉淀的资源。
- 有返回 Today 的入口。

### 3.3 展示资源平台

入口：`/resources`

讲解重点：

- 资源库已经从“生成资源列表”升级为“学习资源平台”。
- 资源来源分为外部视频、官方资料、开放课程、AI 讲解文档、针对练习、个人资料。
- B 站资源只保存元数据和链接，不下载、不转存、不伪装成自有资源。

验收点：

- 资源卡片能显示来源、提供方、预计耗时和推荐理由。
- 外部视频显示外部访问策略。
- 有 `embed_url` 的资源显示“可站内播放”标识。
- 外部链接以新窗口打开。

### 3.4 展示 AI 导师页

入口：`/chat`

讲解重点：

- AI 导师不是客服，也不是通用聊天。
- 它提供 1 对 1 学习诊断、路径规划、练习生成、错题复盘和资源推荐。
- 内部动作 prompt 使用英文，用户界面保持中文。

验收点：

- 页面标题为“1 对 1 AI 学习导师”。
- 空状态包含课程/考试/项目目标、当前基础、期望帮助方式三类引导。
- 有 5 个结构化动作：构建学习画像、生成学习路径、生成一组练习、复盘一道错题、推荐学习资源。
- 不显示旧的通用工具面板。

### 3.5 展示错题复盘

入口：`/mistakes`

讲解重点：

- 错题不是孤立记录，而是学习反馈。
- 错题会影响 Today 建议、学习画像和补救资源。

验收点：

- 能看到错题列表或录入入口。
- 能触发归因、补救计划或针对性资源。
- 页面文案围绕“复盘”和“补救”，不是简单表格管理。

### 3.6 展示成长档案

入口：`/growth`

讲解重点：

- 当前成长档案仍偏协作轨迹。
- 它可以作为系统理解用户和自进化过程的证据。
- 后续需要改成成长趋势页。

验收点：

- 能展示协作轨迹、画像变化或 LoRA 样本导出证据。
- 不声称 LoRA 已训练完成。
- 不把原始 JSON 当作最终产品体验。

## 4. 演示前准备

### 4.1 数据准备

运行：

```bash
bash scripts/seed_demo.sh
```

预期：

- PostgreSQL 可用时写入演示数据。
- PostgreSQL 不可用时输出清晰降级摘要，并以成功状态结束。

### 4.2 启动服务

按项目规则，所有启停必须使用 `scripts/`：

```bash
bash scripts/start_api.sh
bash scripts/start_frontend.sh
```

如需完整启动：

```bash
bash scripts/start_full.sh
```

停止：

```bash
bash scripts/stop_all.sh
```

### 4.3 构建与检查

演示前至少运行：

```bash
bash scripts/check_today_page.sh
bash scripts/check_frontend_ia.sh
bash scripts/build_frontend.sh
bash scripts/test_unit.sh tests/unit/learning/test_today.py
bash scripts/test_unit.sh tests/unit/learning/test_assets.py
bash scripts/test_unit.sh tests/unit/learning/test_demo_seed.py
```

验收标准：

- Today 静态验收通过。
- 前端信息架构检查通过。
- 前端构建通过。
- Today、资源、seed 相关单测通过。

## 5. 页面级验收清单

### 5.1 `/today`

- [ ] 登录后默认进入 `/today`。
- [ ] Today 优先读取 `GET /api/today`。
- [ ] 接口失败时显示“当前显示离线学习建议，稍后会自动恢复同步。”或同等级产品化文案。
- [ ] 主任务有明确标题、原因、预计耗时和行动按钮。
- [ ] 有路径预览。
- [ ] 有推荐资源。
- [ ] 有 AI 导师入口。
- [ ] 有画像信号和复习队列。
- [ ] 主任务有 `spaceId` 时进入 `/spaces/[id]`。

### 5.2 `/spaces/[id]`

- [ ] 使用现有 `GET /api/spaces/{space_id}/detail`。
- [ ] 不新增重复后端接口。
- [ ] 展示学习目标、进度、路径、下一步建议和资源。
- [ ] 不以原始 JSON 或后台表格作为主体验。
- [ ] 有返回 Today 的入口。
- [ ] 降级文案不暴露技术异常细节。

### 5.3 `/resources`

- [ ] 区分外部视频、官方资料、开放课程、AI 讲解文档、针对练习、个人资料。
- [ ] 外部视频不被表述为平台自有资源。
- [ ] B 站资源使用 `metadata_only` 和 `embed_or_redirect_only` 策略。
- [ ] 有 `embed_url` 的资源显示站内播放标识。
- [ ] 外部链接新窗口打开。
- [ ] 页面不是后台表格。

### 5.4 `/chat`

- [ ] 标题和说明明确为 AI 学习导师。
- [ ] 空状态能引导用户输入目标、基础、帮助方式。
- [ ] 有结构化动作入口。
- [ ] 动作 prompt 内部使用英文。
- [ ] 用户界面中文。
- [ ] 不显示旧的通用工具折叠区。

### 5.5 `/mistakes`

- [ ] 能录入或查看错题。
- [ ] 能进行错因归因或补救动作。
- [ ] 错题复盘能被解释为 Today 建议和画像更新的反馈来源。

### 5.6 `/growth`

- [ ] 能作为协作轨迹和自进化证据。
- [ ] 不声称 LoRA 已训练完成。
- [ ] 后续需从技术证据页升级为成长趋势页。

## 6. 不能夸大的内容

演示和页面文案必须遵守：

- 可以说系统具备多 Agent 学习资源生成 MVP。
- 可以说系统具备 Today 聚合、学习空间详情、资源来源表达和 AI 导师入口。
- 可以说 B 站资源以元数据、链接和允许的嵌入方式接入。
- 可以说 seed 数据用于演示结构真实的学习流程。
- 不能说 Google/GitHub 已真实 OAuth。
- 不能说 LoRA 已训练完成。
- 不能说 SeeDance 视频已稳定生成。
- 不能说 B 站、Coursera、MOOC 等外部平台内容是自有资源。
- 不能说已经达到完整生产级教育平台。

## 7. 下一阶段建议

### 7.1 AI 学伴接入

AI 学伴由独立任务推进。接入时必须坚持：

- 它是学习陪伴代理，不是普通圆形聊天按钮。
- 它有形象、有状态、有页面上下文。
- 它可以调用现有 `/api/tutor/ask`。
- 它不应遮挡主要学习流程。

### 7.2 学习画像页

新增或产品化 `/profile`：

- 展示专业方向、学历阶段、目标、薄弱点、偏好、错题模式、能力短板。
- 显示画像来源：对话、路径、错题、资源使用、上传资料。
- 允许用户修正画像。

### 7.3 学习路径页

深化 `/plan`：

- 展示当前路径、阶段、依赖关系、推荐理由。
- 支持调整顺序、标记完成、生成补救节点。
- 和 Today 主任务保持联动。

### 7.4 成长档案页

重构 `/growth`：

- 用趋势、画像变化、薄弱点变化、复习效果表达成长。
- 协作轨迹和 LoRA 样本作为二级证据。

### 7.5 资源详情与真实搜索

后续补齐：

- 资源详情页。
- B 站搜索元数据链路。
- 官方文档 / OER / 论文来源检索。
- 资源收藏、复用、版本和引用来源。
